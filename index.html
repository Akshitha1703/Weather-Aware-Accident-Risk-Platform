üöó Project Overview

The Weather-Aware Accident Risk Platform analyzes the relationship between weather and road accidents across the U.S. using live Power BI dashboards connected through SharePoint.

üéØ Goals

  • Correlate weather factors with accident severity and frequency.
  • Enable dynamic, auto-refreshing reports via SharePoint.
  • Provide clear, actionable visual insights for safety planning.

  • Microsoft Power BI Desktop

üì¶ Data Source & Dynamic Connection

Data sourced from US Accidents (Updated), containing 7M+ records with location, time, and weather details.

  1. Download dataset from Kaggle.
  2. Clean/filter in Excel Power Query.
  3. Split into Accident_Data and Weather_Data.
  4. Upload to SharePoint for web URL link.
  5. Use Get Data ‚Üí Web in Power BI to connect live.

üßÆ Data Preparation & Transformation

  • Removed unnecessary columns (Amenity, Bump, Junction etc.).
  • Derived columns: Year, Month, Day_of_Week, Hour, GeoKey (City + State + Zip).
  • Validated duplicates & null values in Power Query Editor.
  • Final files uploaded to SharePoint for dynamic connection.

üß† Data Model & DAX Calculations

Data modeled in a star schema with fact and lookup tables.

TableDescription
Accident_DataFact table with accident details
Weather_DataLookup table for weather parameters
Dim_DateCalendar created via CALENDAR()
Dim_HourHour lookup (0‚Äì23)

DAX Measures

  • Total Accidents = COUNTROWS(Accident_Data)
  • Avg Temperature = AVERAGE(Weather_Data[Temperature(F)])
  • Avg Visibility = AVERAGE(Weather_Data[Visibility(mi)])
  • Accidents by Year = CALCULATE(COUNTROWS(Accident_Data), VALUES(Dim_Date[Year]))
  • Weather Impact Index = Custom ratio of accidents to precipitation & visibility
